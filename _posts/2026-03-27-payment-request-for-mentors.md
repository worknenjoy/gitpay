## What a payment request means (for Mentors)

A payment request is a simple link you send after delivery so the client can pay immediately—without attaching PDFs, chasing signatures, or waiting for an accounting workflow.

## The typical payment delay loop

- Deliver work
- Send invoice
- Wait
- Follow up

## A faster workflow with a payment request link

1. Deliver the milestone
2. Send one clear payment request link
3. Client forwards it internally if needed
4. Payment happens faster because the action is obvious

## Example use cases for Mentors

- Mentorship hour block
- Career review session
- Code / portfolio review
- Ongoing monthly mentorship

## Why payment requests work better than invoices

- A link is easier to approve and pay
- Less friction for small and medium deliverables
- Works well with international clients
