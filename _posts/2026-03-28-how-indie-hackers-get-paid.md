## How Indie Hackers get paid (in practice)

Most payment delays aren’t about the work quality—they’re about unclear payment steps. A clean workflow makes “pay now” the default next action.

## The old workflow

1. Deliver
2. Invoice
3. Wait
4. Follow up

## The modern workflow (payment requests)

1. Deliver the milestone
2. Send a payment request link
3. Client pays (or forwards internally)

## Example deliverables

- MVP build for a client
- Setup fee for onboarding
- One-off feature build
- Support + maintenance add-on
