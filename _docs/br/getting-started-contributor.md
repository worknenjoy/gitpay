## O que este guia cobre
Esta página ajuda você a sair do “encontrei uma tarefa” até “meu trabalho foi aceito e fui recompensado”.

## Fluxo básico
1. Escolha uma tarefa que você consegue entregar.
2. Demonstre interesse / se candidate.
3. Seja escolhido (ou aprovado).
4. Implemente a mudança em um branch (normalmente via fork).
5. Envie um Pull Request.
6. Ajuste o que for pedido e passe na CI.
7. Após aceitação/merge, o pagamento é liberado conforme as regras do projeto.

## Antes de começar
- Garanta que você consegue rodar o projeto localmente (se necessário).
- Confirme critérios de aceitação na issue (passos de teste, casos de borda, screenshots).
- Combine o formato de entrega (PR, patch, link, etc.).
