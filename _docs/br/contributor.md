## Para quem é esta página
Este guia é para quem quer trabalhar em tarefas publicadas no Gitpay (issues) e ser recompensado depois que o trabalho é revisado e aceito.

## Fluxo típico
1. Escolha uma tarefa que você consegue entregar.
2. Demonstre interesse e se candidate.
3. Implemente a correção/funcionalidade em um fork ou branch.
4. Envie um Pull Request.
5. Ajuste o que for pedido no review.
6. Após a aceitação/merge, o Gitpay libera o pagamento da recompensa.

## O que significa “concluído”
No Gitpay, a conclusão está ligada ao fluxo do Git: a tarefa é validada pelo processo do repositório (review, CI, merge) e então o pagamento é liberado.
