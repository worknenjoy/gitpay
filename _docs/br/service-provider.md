## Para quem é esta página
Este guia é para freelancers, consultores e agências que entregam trabalho em repositórios Git e querem um processo simples e rastreável para receber.

## Como o Gitpay ajuda
- Conecta entregas ao histórico do Git (issues, commits, pull requests).
- Torna a aceitação explícita via review e merge.
- Reduz discussões de pagamento ao alinhar critérios de aceitação antes.

## Fluxo recomendado
1. Defina a entrega e os critérios de aceitação.
2. Trabalhe em um repositório com um processo claro de Pull Request.
3. Entregue via Pull Request.
4. Use a aceitação/merge como sinal de validação para liberar o pagamento.
