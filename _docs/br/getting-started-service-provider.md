## O que este guia cobre
Esta página é para quem entrega trabalho pago (serviços) usando o repositório para validar a entrega.

Use este guia quando precisar de evidência de entrega rastreável antes de receber o pagamento (por exemplo: tarefas de desenvolvimento, revisões de design, atualizações de conteúdo ou artefatos de consultoria técnica).

## Fluxo típico de entrega
1. Combine escopo e critérios de aceitação.
2. Entregue em PRs pequenos quando possível.
3. Valide via review + CI.
4. Marque como aceito quando os critérios forem atendidos.
5. Dispare o pagamento com base na aceitação.

## Definir termos de entrega antes de começar
Alinhe com o cliente sobre:
- Entregáveis e exclusões.
- Marcos e prazos.
- Número de rodadas de revisão incluídas.
- Responsável pela aprovação (quem assina).
- Gatilho de pagamento (por marco ou aceitação total).

## Construir confiança com progresso transparente
- Compartilhe atualizações curtas nos checkpoints acordados.
- Vincule commits/PRs a cada marco.
- Sinalize riscos cedo (mudança de escopo, dependências bloqueadas, alterações de prazo).
- Mantenha todas as decisões de aprovação e escopo documentadas.

## Checklist pré-entrega
1. Confirme que todos os critérios de aceitação estão cobertos.
2. Certifique-se de que os assets/documentos solicitados estão anexados.
3. Verifique checks e testes relevantes ao escopo.
4. Prepare um resumo conciso do que foi entregue.
5. Confirme a janela de review do cliente e a próxima ação.
