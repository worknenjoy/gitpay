## Who this page is for
This guide is for freelancers, consultants, and agencies who deliver work in Git repositories and want a simple, traceable way to request and receive payment.

## How Gitpay helps
- Connect deliverables to Git history (issues, commits, pull requests).
- Make acceptance explicit via review and merge.
- Reduce payment disputes by agreeing on acceptance criteria up front.

## Recommended workflow
1. Define the deliverable and acceptance criteria.
2. Work in a repository with a clear Pull Request process.
3. Deliver via Pull Request.
4. Use the accepted/merged work as the validation signal for payment.
